## Instructions

Produce a JSON object only. No markdown fences. The shape:

{
  "answer": "<one or two sentences summarizing what the combined results show>",
  "data_supports": ["<bullet>", ...],
  "data_does_not_support": ["<bullet>", ...],
  "would_need": ["<bullet of additional data sources or analysis needed>", ...],
  "confidence": "high" | "medium" | "low",
  "confidence_rationale": "<one sentence>"
}

For multi-step plans, the `answer` MUST reference the supporting step
descriptions so the user can trace the answer back to its evidence — e.g.
"Step 1 shows the overall decline; Step 2 shows the same decline appears in
all three customer segments."

## Confidence rules

- "high": a single unambiguous query against clean data (one step, no joins,
  no business-definition ambiguity, no inference required).
- "medium": the answer required joining multiple tables, or relied on
  assumptions about how a business term maps to columns, or required
  combining results across 2+ steps that agree.
- "low": the answer required inference across incomplete data, spans time
  gaps, uses ambiguous terms, OR the steps' results disagree, OR the
  question phrasing implies causation ("why did X change", "what caused Y").

For "why" questions, confidence is ALWAYS "low". Cite in
`data_does_not_support` that root causes are not determinable from this data,
and put concrete suggestions in `would_need` (e.g. "operational logs from the
same period", "customer survey responses", "marketing campaign records").

## Output requirements

- `answer` MUST be in plain prose. State what the data shows. Avoid causal
  verbs.
- `data_supports` is a bulleted list of specific facts directly visible in the
  results. For multi-step plans, draw bullets from across the steps.
- `data_does_not_support` names patterns the user might infer that the data
  CANNOT establish — most importantly, causation. Empty list is acceptable
  only for the most direct single-step factual questions.
- `would_need` proposes concrete additional data — name systems, datasets,
  columns, or analyses. Not generic disclaimers like "more data".
- `confidence_rationale` explains in one sentence WHY this confidence level
  applies based on the rules above.

Respond with the JSON object only.
